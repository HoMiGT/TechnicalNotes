# 一、自定义损失函数
PyTorch在torch.nn模块中提供了许多常用的损失函数    
自定义损失函数
## 1. 以函数方式定义
损失函数仅仅是一个函数而已， 因此可以通过直接以函数定义的方式定义一个自己的函数
```Python
def my_loss(output, target):
    loss = (output - target).abs().mean()
    return loss
```
## 2. 以类方式定义
以函数的定义更简单，但以类的定义更加常用，以类定义损失函数时，我们看到每一个损失函数的继承关系
```Python
class DiceLoss(nn.Module):
    def __init__(self, weight=None, size_average=True):
        super(DiceLoss, self).__init__()
    def forward(self, inputs, targets, smooth=1):
        inputs = F.sigmoid(inputs)
        inputs = inputs.view(-1)
        targets = targets.view(-1)
        intersection = (inputs * targets).sum()
        dice = (2. * intersection + smooth) / (inputs.sum() + targets.sum() + smooth)
        return 1 - dice
criterion = DiceLoss()
loss = criterion(inputs, targets)
```
除此之外， 常见的损失函数还有BCE-DiceLoss、IoU Loss、Focal Loss等， 这些损失函数都是以类的方式定义的。
```Python
class DiceBCELoss(nn.Module):
    def __init__(self, weight=None, size_average=True):
        super(DiceBCELoss, self).__init__()
        
    def forward(self, inputs, targets, smooth=1):
        inputs = F.sigmoid(inputs)
        inputs = inputs.view(-1)
        targets = targets.view(-1)
        intersection = (inputs * targets).sum()
        dice_loss = 1 - (2. * intersection + smooth) / (inputs.sum() + targets.sum() + smooth)
        BCE = F.binary_cross_entropy(inputs, targets, reduction='mean')
        return BCE + dice_loss
```
```Python
class IoULoss(nn.Module):
    def __init__(self, weight=None, size_average=True):
        super(IoULoss, self).__init__()
        
    def forward(self, inputs, targets, smooth=1):
        inputs = F.sigmoid(inputs)
        inputs = inputs.view(-1)
        targets = targets.view(-1)
        intersection = (inputs * targets).sum()
        total = (inputs + targets).sum()
        union = total - intersection
        IoU = (intersection + smooth) / (union + smooth)
        return 1 - IoU
```
```Python
ALPHA = 0.8
GAMMA = 2 
class FocalLoss(nn.Module):
    def __init__(self, weight=None, size_average=True):
        super(FocalLoss, self).__init__()

    def forward(self, inputs, targets, alpha=ALPHA, gamma=GAMMA):
        inputs = F.sigmoid(inputs)
        inputs = inputs.view(-1)
        targets = targets.view(-1)
        BCE = F.binary_cross_entropy(inputs, targets, reduction='mean')
        BCE_EXP = torch.exp(-BCE)
        focal_loss = alpha * (1 - BCE_EXP) ** gamma * BCE
        return focal_loss
```
在定义的损失函数时，涉及到数学运算时，最好全程使用PyTorch提供的张量计算接口，
这样就不需要我们实现自动求导功能并且我们可以直接调用cuda，使用numpy或scipy的数学运算时，
操作会有些麻烦。

# 二、动态调整学习率
学习率的选择是深度学习中一个困扰人们许久的问题，学习速率设置过小，会极大降低收敛速度，增加训练时间；
学习率太大，可能会导致参数在最优解两侧来回震荡。
## 1. 使用官方Scheduler 
PyTorch提供了torch.optim.lr_scheduler模块，提供了多种学习率调整策略，常用的有：
- lr_scheduler.LambdaLR
- lr_scheduler.MultiplicativeLR
- lr_scheduler.StepLR
- lr_scheduler.MultiStepLR
- lr_scheduler.ExponentialLR
- lr_scheduler.CosineAnnealingLR
- lr_scheduler.ReduceLROnPlateau
- lr_scheduler.CyclicLR
- lr_scheduler.OneCycleLR
- lr_scheduler.CosineAnnealingWarmRestarts
- lr_scheduler.ConstantLR
- lr_scheduler.LinearLR
- lr_scheduler.PolynomialLR
- lr_scheduler.ChainedScheduler
- lr_scheduler.SequentialLR
## 2. 自定义scheduler
```Python
def adjust_learning_rate(optimizer, epoch):
    lr = args.lr * (0.1 ** (epoch // 30))
    for param_group in optimizer.param_groups:
        param_group['lr'] = lr
optimizer = torch.optim.SGD(model.parameters(), lr=args.lr, momentum=0.9)
for epoch in range(100):
    train(...)
    validate(...)
    adjust_learning_rate(optimizer, epoch)
```

## 1. torchvision
收集和标注数据会花费大量的时间和资金。     
迁移学习，将从源数据学习到的知识迁移到目标数据集上。     
迁移学习的一大应用场景是模型微调。     
在PyTorch中提供了许多预训练好的网络模型(VGG、ResNet、DenseNet、Inception等)，我们可以直接使用这些预训练好的模型进行微调。
### 1. 模型微调的流程
1. 在源数据集上预训练一个神经网络模型，即源模型
2. 创建一个新的神经网络模型，即目标模型。 它复制了源模型上除了输出层外的所有模型设计及其参数。
3. 为目标模型添加一个输出大小为目标数据集类别个数的输出层，并随机初始化该层的模型参数。
4. 在目标数据集上训练目标模型。

# 四、半精度训练
GPU的性能主要分为两部分：算力和显存。    
算力决定了显卡计算的速度。    
显存决定了同时放入多少数据用于计算。    

PyTorch默认存储方式用的是torch.float32, 也就是单精度浮点数。但大多数场景其实并不需要这么精确，只保留一半的信息也不会影响结果
即使用torch.float16, 也就是半精度浮点数。

## 1. 半精度训练的设置
在PyTorch中，使用autocast配置半精度训练，同时需要在下面三处加以设置:
* import autocast
```
from torch.cuda.amp import autocast
```
* 模型设置
```
@autocast()
def forward(self, x):
    ...
    return x 
```
* 训练过程
```
for x in train_loader:
    x = x.cuda()
    with autocast():
        output = model(x)
```
注意：
    半精度训练主要适用于数据本身的size比较大。当数据本身的size并不大时，使用半精度训练则可能不会带来显著的提升。

## 1. 核心概念
### 1.1 变换
关键概念:
* 单次操作：每次变化执行一次特定的增强操作
* 概率控制：p参数控制应用可用性(0.0-1.0)
* 参数抽样：每次从指定范围内选择随机值
* 变化类型：像素变换和空间变换
* 目标兼容性：某些变换会影响多种数据类型(图像、掩码、边界框、关键点)

首先要进行的最重要转换：
* 调整大小/裁剪: A.RandomCrop, A.RandomResizedCrop
* 基本几何图形: A.HorizontalFlip
* 正则化：A.CoarseDropout
* 缩放/旋转：A.Affine

常见用法模式：
* 经常应用： A.Resize(height=224, width=224, p=1.0)
* 偶尔应用： A.HorizontalFlip(p=0.5)
* 随机范围: A.RandomBrightnessContrast(brightness_limit=0.2, contrast_limit=0.2, p=0.8)

#### 1.1.1 基本转换用法    

应用单个变换
```
import albumentations as A
import cv2
import numpy as np 

image = np.random.randint(0, 256, (100, 100, 3), dtype=np.uint8)
transform = A.HorizontalFlip(p=1.0)
transformed_data = transform(image=image)
transformed_image = transformed_data['image']
print(f"Original shape: {image.shape}, Transformed shape: {transformed_image.shape}")
```

所有变换都会返回一个字典，而不仅仅是变换后的图像：
```

transformed_image = result['image']

result = transform(image=image, mask=mask)
new_image = result['image']
new_mask = result['mask']
```

#### 1.1.2 核心转换概念    
概率参数p    
每个变换都有一个关键参数p，控制着调用变换时应用该变换的概率    
概率值：
* p=1.0 变换始终应用
* p=0.0 变换永远不应用
* p=0.5 变换有50%的概率应用
```
import albumentations as A
always_flip = A.HorizontalFlip(p=1.0)
maybe_flip = A.HorizontalFlip(p=0.5)
never_flip = A.HorizontalFlip(p=0.1)
```
这种随机性使你能够在数据增强流程中引入多样性

参数采样和范围     
除了p概率之外，许多变换还会通过接受某些参数的值范围(min_value, max_value)
来引入变异性
```
import albumentations as A
brightness_contrast = A.RandomBrightnessContrast(
    brightness_range=(-0.2, 0.3), 
    contrast_range=(-0.1, 0.1), 
    p=1.0)
rotation_transform = A.Rotate(
    angle_range=(-15, 15),
    p=0.7
)
fixed_blur = A.GaussianBlur(blur_range=(3, 3), p=1.0)
random_blur = A.GaussianBlur(blur_range=(1, 5), p=1.0)
```
关键点： 每次调用变换时，都会从范围内采样新的随机值，即使使用同一个变换实例，也会产生不同的变化。

#### 1.1.3 转换类别    
##### 1.1.3.1 技术类别    
像素变换     
仅修改图像本身的像素值，不改变图像的几何形状或空间排列     
对目标的影响：这些变换仅影响图像类目标(图像、多图像集、体数据)。不会修改掩码、边界框或关键点。    
常见的像素变换:
* 颜色调整： 随机亮度对比度、色相饱和度、颜色抖动
* 模糊效果：高斯模糊，运动模糊，散焦
* 噪声: 高斯噪声、ISO噪声、乘性噪声
* 压缩：图像压缩

空间变换    
修改图像的空间属性——其几何形状、大小或方向
对目标的影响：由于这些改变会改变几何形状，因此会影响所有空间目标：图像、掩膜、边界框、关键点和体数据。所有目标都会进行一致的变换以保持对齐    
常见的空间变换：
* 翻转：水平翻转、垂直翻转
* 旋转：旋转、安全旋转
* 调整大小：调整大小、随机缩放、随机裁剪
* 几何变形：仿射变换、透视变换、弹性变形
* 裁剪方式：随机裁剪、居中裁剪、BBoxSafe随机裁剪

##### 1.1.3.2 实用分类
1. 基本要素
**始终从以下几点开始**：
* 尺寸归一化: 随机裁剪、随机调整大小裁剪、最小最大尺寸
* 基本不变性：水平翻转(对大多数自然图像安全)
```
essential_pipeline = A.Compose([
    A.RandomCrop(height=224, width=224, p=1.0),
    A.HorizontalFlip(p=0.5)
])
```

2. 高影响力正则化(添加下一步)    
已被证实能提高泛化能力：
* Dropout变体：CoarseDropout、Erasing 
* 缩放/旋转: 仿射(先考虑保守范围)

improved_pipeline = A.Compose([
    A.RandomCrop(height=224, width=224, p=1.0),
    A.HorizontalFlip(p=0.5),
    A.CoarseDropout(num_holes_range=(1, 8), hole_height_range=(0.1, 0.15), hole_width_range=(0.1, 0.15), p=0.5),
    A.Affine(scale=(0.8, 1.2), rotate=(-15, 15), p=0.7),
])

3. 特定领域增强
对具体旋转对称性的航拍/医学图像：
   * 正方形对称： 应用正方形的所有8种旋转/翻转方式
对于对颜色敏感的任务
   * RandomBrightnessContrast: 模拟光照变化
   * ColorJitter: 全面的色彩增强
为了提高对模糊/噪声的鲁棒性：
   * 高斯模糊：相机焦点变化
   * 高斯噪声：传感器噪声模拟
为了降低对颜色的依赖性
   * ToGray: 力形状/纹理学习
   * 通道丢失：部分颜色移除

4. 高级/专业级(谨慎使用)
针对特定领域或基本增强功能不足以满足需求的情况：
    * 医学成像：弹性变换、网格畸变
    * 天气模拟：随机太阳耀斑、随机降雨、随机雾
    * 领域自适应：FDA、直方图匹配

按照以下顺序逐步构建您的管道：
1. 尺寸标准化——先裁剪或调整大小(始终如此)
2. 基本几何不变性——水平翻转、方形对称(适用于航空/医疗领域)
3. Dropout/遮挡——CoarseDropout、ConstrainedCoarseDropout 影响巨大!
4. 降低颜色依赖性——ToGray、通道丢弃(如需要)
5. 仿射变换——用缩放/旋转的仿射变换
6. 领域特定——针对您的用例的专用转换
7. 归一化——标准或样本特定(始终最后进行)

依赖链： 
    分辨率 -> 几何形状 -> 遮挡 -> 颜色 -> 域变化 -> 归一化     
每一步都依赖前一步的完整

没有一个公式能够直接给出最优的数据增强流程。     
但存在一个可靠的方法，可以让你获得一个相当不错的流程     
需要记住三点：
1. 先从问题入手，而不是从变换入手。我的模型需要对哪些训练数据未涵盖的情况保持不变？    
2. 精准测量至关重要，汇总指标会误导人。按类别细分、在特定条件下进行鲁棒性测试以及按条件进行切片，才是区分外观良好的流程和真正能在生产环境中有效运行的流程的关键所在。

#### 1.2.4 管道中概率是如何运作的
当调用使用以下方式创建的管道时 A.Compose:
1. 它按照提供的顺序遍历列表中的变换
2. 对列表中的每个p变换，检查其各自的值
3. 如果对特定变换的随机检查通过(基于其p)，则该变换将应用于数据的当前状态
4. 如果检查失败，则跳过该特定转换，管道继续执行下一个转换

这意味着在对管道的任何给定调用中，实际应用的可能是已定义变换的不同子集，由它们各自的概率控制

> **重要提示：** A.Compose本身没有顶级p参数来控制整个管道是否应用。根据具体p值来决定是否激活它们     
> 如果要将变换分组到单个概率下，请改用 Sequential

部分应用变换的随机子集：    
A.SomeOf: 接受一个变换列表和一个数字n。     
主要特点： 
* 转换数量（n）：这可以是固定的整数或元组。如果是元组，则每次调用时选择的(min_n, max_n)转换数量将是在该范围内均匀随机选择的整数（包含该范围和整数）。[min_n, max_n]
* 选择方法（replace）：默认情况下（replace=False），变换选择不放回。如果replace=True指定了选择方法，则变换选择有放回。
* 均匀选择：变换从列表中均匀随机选择——每个变换被选中的机会均等，无论其具体p值如何。
* 主概率( p)：控制SomeOf代码块执行的概率。如果此检查失败，则不会考虑其中的任何转换。
* 个体概率：至关重要的是，在SomeOf选择转换后，它会尝试应用每个转换。每个选定的转换只有在其自身概率p值满足条件时才会应用。p列表中的值在选择后不会被忽略。
```
import albumentations as A

pipeline = A.Compose([
    # Attempt to apply between 1 and 3 of the following transforms (inclusive).
    # Selection is without replacement (default).
    # This SomeOf block has a 90% chance of being entered.
    A.SomeOf([
        A.GaussianBlur(p=0.5), # If selected, 50% chance to apply
        A.RandomBrightnessContrast(p=1.0), # If selected, 100% chance to apply
        A.HueSaturationValue(p=0.8), # If selected, 80% chance to apply
        A.Sharpen(p=0.3), # If selected, 30% chance to apply
    ], n=(1, 3), p=0.9), # Select 1 to 3 transforms, 90% of the time
    A.HorizontalFlip(p=0.5)
])

# When pipeline(image=...) is called:
# - 90% chance:
#     1. SomeOf selects N transforms (N between 1 and 3) randomly from the list.
#     2. It then iterates through these N selected transforms.
#     3. For each selected transform, it applies it *only if* that transform's individual `p` value passes its random check.
# - 10% chance: None of the transforms in SomeOf are considered.
# - Independently, 50% chance HorizontalFlip is applied.
```

### 1.3 使用目标(图像、掩码、边界框、关键点)
#### 1.3.1 快速参考
关键概念: 
* image或images: 必填-主要输入数据
* 空间变换：影响所有目标(图像、掩膜、边界框、关键点、体数据)
* 像素变换：仅影响图像类目标(图像、多图像、体数据)
* 同步增强：所有目标接受相同的空间变换
* 格式配置：bbox_params以及keypoint_params各个目标所需的配置

常见靶点组合：
* 分类 image
* 分割 image + mask
* 目标检测 image + bboxesbbox_params
* 关键点检测 image + keypointskeypoint_params 
* 3D数据处理 volume + mask3d 

#### 1.3.2 核心要求
**数据类型要求**      
传递给Alumentations的所有数据必须是NumPy数据：    
* 图像数据(image, images, volume, volumes): 必需uint8是float32的Numpy数组
* 掩码: 可以是任何整数类型的Numpy数组
* 边界框: 必须是Numpy数组，通常float32具有特定形状
* 要点：必需是Numpy数组，通常是float32具有特定形状
* 标签: 必须是Numpy数组

**灰度图像处理**     
Compose会自动进行处理灰度图像，方便使用      
自动通道尺寸管理：
* 输入灵活：可以传递带或不带有透明通道维度的灰度图像
* 内部预处理：Compose如果缺少通道维度，则自动添加:
  * (H, W) -> (H, W, 1)
  * (N, H, W) -> (N, H, W, 1) 用于多张图片
  * (D, H, W) -> (D, H, W, 1) 用于卷
* 一致性处理： 所有变换均在一致性的维度上进行操作
  * 单张图片： ndim=3
  * 多张图片： ndim=4
  * 多卷本： ndim=5
* 自动清理： 如果添加了通道维度，则Compose在后处理过程中将其删除，恢复原始格式

```
import albumentations as A
import numpy as np 
transform = A.Compose([
    A.HorizontalFlip(p=1.0),
])

grayscale_2d = np.random.randint(0, 256, (100, 100), dtype=np.uint8)
grayscale_3d = np.random.randint(0, 256, (100, 100, 1), dtype=np.uint8)

result_2d = transform(image=grayscale_2d)
result_3d = transform(image=grayscale_3d)

flip = A.HorizontalFlip(p=1.0)

grayscale_with_channel = grayscale_2d[..., np.newaxis]  # (100, 100) -> (100, 100, 1)
flipped = flip(image=grayscale_with_channel)
```

### 1.4 设置概率
#### 1.4.1 快速参考
**关键概念:**
* p=1.0 转换始终是应用过程中需要考虑的因素。
* p=0.0 转换从未被考虑用于应用程序。
* p=0.5 转换有50%的概率被考虑
* 管道运行概率：整个管道运行的总体概率
* 嵌套概率：概率如何在组合块中组合, 例如OneOf

#### 1.4.2 基本概率力学

#### 1.4.4 不同概率等级是如何运作的
管道概率(prob_pipeline)    
该p参数A.Compose决定是否对其中的任何增强应用：
* p=0.0管道从未运行，输入始终原封不动地返回。
* p=1.0管道始终运行，内部变换根据各自的概率获得执行机会
* 0 < p < 1 管道运行具有该特定概率 

在我们的示例中，(prob_pipeline=0.95), 管道运行时间为95%  

#### 1.4.5 OneOf中的概率计算
选择机制如何运作    
当一个OneOf代码块运行时，它会对内部变换的概率进行归一化，以确定选择权重：
* GaussNoise(p=0.9)
* ISONoise(p=0.7)

标准化过程：
1. 概率之和: 0.9 + 0.7 = 1.6 
2. 高斯噪声的归一化概率: 0.9 / 1.6 = 0.5625 (56.25%机会被选中)
3. ISO噪声的归一化概率: 0.7 / 1.6 = 0.4375 (43.75%机会被选中)

#### 1.4.6 总体概率计算

|   转换   |          计算          |  最终概率  |
|:------:|:--------------------:|:------:|
| 随机旋转90 |     0.95 x 0.85      | 80.75% |
|  高斯噪声  | 0.95 x 0.75 x 0.5625 | 40.08% |
| ISO噪声  | 0.95 x 0.75 x 0.4375 | 31.15% |

#### 1.4.7 最佳实践
**概率设定指南**
1. 务必明确设置值：始终p明确设置值，而不是依赖默认值
2. 考虑独立性：记住，变换概率在范围内是独立的 
3. 计算总体影响：使用乘法规则来理解最终概率
4. 测试您的流程：验证您的概率设置是否达到所需的增强频率

### 2.2 选择增强数据以提高泛化能力
#### 2.2.1 为什么增强技术需要严谨的工程设计
增强技术实际上是对神经网络设计根本局限性的一种有原则的回应。     
有些不变性可以直接编码到网络架构中。     
大多数现实世界的不变性并非纯粹的数学对称性。 不存在"雾等变卷积"    
数据增强策略应该与框架选择、损失函数设计或优化器调优一样严谨。是模型学习泛化能力的核心组成部分。     
这个严谨始于一个问题，应该考虑添加的每一个变换提出这个问题。

#### 2.2.2 核心思想：每个变换都是一种不变性断言
根本问题: 不在于"我应该使用哪些变换?"     
而在于"我的模型需要学习哪些不变性，以及我的训练数据中哪些不变性没有得到充分体验？"      
你添加的每一个变换都隐含着一个声明："无论这种变化如何，我的模型都应该产生相同的输出。"      
不变性主张框架告诉我们应该针对每个变换提出什么问题，接下来的问题是，我们应该将这种主张推向何种程序——这取决于变换服务于俩个截然不同的目的中的哪一个。

#### 2.2.3 俩种增强级别
* **第一级: 你未收集到的可能变体**
* **第二级: 刻意增加难度以增强功能**

唯一限制：     
俩层都遵循一条不可妥协的规则： 转换后标签必须保持明确无误。

#### 2.2.4 快速参考：七步法
按以下顺序逐步构建您的管道：
1. 尺寸标准化——先裁剪或调整大小(始终如此)
2. 基本几何不变性——水平翻转、方形对称
3. Dropout/遮挡——CoarseDropout, ConstrainedCoarseDropout(影响巨大)
4. 降低颜色依赖性——ToGray, 通道丢弃
5. 仿射变换——用于缩放、旋转的仿射变换
6. 领域特定——针对特定用例的专用转换
7. 归一化——标准或样本特定(始终最后进行)

```
class ImageClassificationDataset(Dataset):
    def __init__(self, image_paths, labels, transforms=None):
        self.image_paths = image_paths
        self.labels = labels
        self.transforms = transforms

    def __len__(self):
        return len(self.image_paths)

    def __getitem__(self, idx):
        image = cv2.imread(self.image_paths[idx])
        image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
        label = self.labels[idx]

        if self.transforms:
            augmented = self.transforms(image=image)
            image = augmented['image']
            
        return image, label
```

关于掩模插值和目标处理的重要说明：      
在应用几何变换(如调整大小、旋转或扭曲)时，用于计算新像素值(插值)的方法至关重要，尤其是对于蒙版而言。    
* 类别掩码(分割): 对于典型的分割掩码，像素值代表离散的类别ID(如 0=背景，1=汽车，2=人)，使用双线性插值或双三次插值等插值方法可能会产生无效的像素值。如在类别ID为0的像素和ID为2的像素之间进行插值，可能会得到一个值为1的像素，从而错误地引入一个原本不存在的类别。为了避免这种情况，需要使用最近邻插值(cv2.INTER_NEAREST)
* 连续目标(深度图、热图)：对于目标为连续体的任务(例如深度图或热图)，通常需要进行值插值。cv2.INTER_LINEAR
* 图像复原用例(图像作为掩码): 图像复原任务中出现了一种有趣的模式。将同一张干净图像同时作为image和mask参数传递给Compose。几何增强将对两者应用相同的操作。然后，在流程中加入仅针对图像增强(A.GaussNoise, A.GaussianBlur)。这些增强只会影响作为image，而不会影响mask。输出字典将包含一张噪声/模糊图像image和一张干净且几何对齐的图像mask，从而为训练复原模型提供完美的输入/目标图像对

```
denoise_transform = A.Compose([
```

```
        mask_path = self.mask_paths[idx]

            image = augmented['image']
            mask = augmented['mask']
        return image, mask
```

### 2.6 目标检测(边界框)
在进行图像增强以进行目标检测时，边界框坐标必须与像素同步变换。水平翻转会镜像图像，但如果边界框坐标保持不变，则每个边界框现在都指向错误的目标。
Albumentations可以自动处理这种情况：只需声明边界框格式，每次空间变换都会同时更新像素和坐标。

#### 2.6.1 边界框格式
|       格式       |                     坐标                      |   价值观    |         常见于         |
|:--------------:|:-------------------------------------------:|:--------:|:-------------------:|
|   pascal_voc   |        [x_min, y_min, x_max, y_max]         |    像素    | PASCAL_VOC,许多自定义数据集 |
| albumentations |        [x_min, y_min, x_max, y_max]         | 归一化[0,1] |       内部相册格式        |
|      coco      |    [x_min, y_min, box_width, box_height]    |    像素    |       COCO数据集       |
|      yolo      | [x_center, y_center, box_width, box_height] | 归一化[0,1] | Ultralytics Yolo,暗网 |
|     cxcywh     | [x_center, y_center, box_width, box_height] |    像素    |    类似YOLO,但并不正常化    |

#### 2.6.4 BboxParams参考
BboxParams参考
* format: 格式
* bbox_types: 'hbb'轴对齐 4个坐标 默认, 'obb' 定向 5个坐标 带角度
* label_fields: 每个框关键字参数名称列表
* min_area: 增强后的最小像素面积。较小的区域将被舍弃。默认是0.0 
* min_visibility: 必须保持可见的原始框面积的最小比率(0.0-1.0)。默认是0.0 
* min_width: 最小框宽度。 默认0.0 
* min_height: 最小框高度。 默认0.0
* clip_bboxes_on_input: 在数据增强之前，将坐标裁剪到图像边界，这对于超出图像范围的标注非常有用。 默认值 False 
* filter_invalid_bboxes: 在数据增强之前移除无效框
* max_accept_ratio: 最大宽高比。 超过此值的方框将被丢弃。None禁用。默认None

处理不完善的注释    
使用clip_bboxes_on_input=True可以在数据增强之前强制坐标位于边界内，并filter_invalid_bboxes=True在裁剪后丢弃掉任何退化(宽度/高度为零)的方框
```

    label_fields=['class_labels'],

    filter_invalid_bboxes=True,
```

min_area丢弃绝对尺寸过小的框

#### 2.6.5 目标检测的裁剪策略
RandomCrop 会生成不包含任何边界框的裁剪区域    
* AtLeastOneBBoxRandomCrop 保证裁剪区域至少包含一个框，但可能会丢失一些框。适用于图像包含大量对象且需要不同裁剪效果的情况。
* BBoxSafeRandomCrop 保证所有框都保留。裁剪区域会自动调整以保留每个框。适用于任何标注丢失
* RandomSizedBBoxSafeCrop 在保留所有边界框的情况下，随机裁剪图像的一部分，然后将其调整为目标尺寸。它可在保留所有对象的同时，提供缩放比例和宽高比的变化——检测训练中最常用的选择

### 2.7 定向边界框(OBB)
定向边界框(OBB)除了轴对齐边界框(HBB)使用的四个坐标外，还增加了一个第五个坐标——角度，用于表示定向物体。      
OBB适合航拍图像、船舶、车辆以及任何需要关注物体方向的场景。

#### 2.7.1 HBB vs OBB
* HBB(轴对齐): 4个坐标——[x_min, y_min, x_max, y_max]或其他格式的等效坐标。框的边缘与图像轴平行
* OBB(定向): 5个坐标——与HBB相同的4个坐标 + 1个角度，该盒子可以旋转

#### 2.7.2 OBB的所有5种格式
|       格式       |             HBB              |               OBB               |
|:--------------:|:----------------------------:|:-------------------------------:|
|      coco      |     [x_min, y_min, w, h]     |   [x_min, y_min, w, h, angle]   |
|   pascal_voc   |  [x_min,y_min,x_max,y_max]   | [x_min,y_min,x_max,y_max,angle] |
| albumentations | [x_min,y_min,x_max,y_max]归一化 | [x_min,y_min,x_max,y_max,angle] |
|      yolo      |        [cx,cy,w,h]归一化        |        [cx,cy,w,h,angle]        |
|     cxcywh     |        [cx,cy,w,h]像素         |        [cx,cy,w,h,angle]        |

#### 2.7.3 角度语义
角度语义    
该角度是宽度边缘相对于水平方向的旋转角度，范围为[-90,90)度     
* 宽度=平行于水平方向的边缘长度
* 高度=另一侧边的长度
* 角度=宽度边缘相对于水平方向的旋转角度

矩形的四个角点计算 [-w/2,-h/2], [w/2,-h/2], [w/2,h/2], [-w/2,h/2]    
0°时，宽度边为水平方向  

#### 2.7.4 OBB的BboxParams 
设置bbox_type='obb'并选择format

#### 2.7.5 转型支持
大多数变换都支持OBB, 包括仿射变换、旋转变换、水平翻转变换和垂直翻转变换。如果某个变换不支持OBB, 则管道将在初始化时引发错误

#### 2.7.6 OBB剪辑
* clip_bboxes_on_input: 在流水线处理之前，将图像框裁剪到图像边界一次。对于OBB格式，此操作会造成图像损失——角点超出[0,1]范围的框会变为轴对齐(角度=0)。建议在仿射变换或旋转变换时对OBB格式进行此操作，设置 False。
* clip_after_transform: 每次变换后进行裁剪。对于OBB格式，裁剪所有4个旋转角点，并返回一个包裹的轴对齐立方体(角度=0)。 默认值: True。 False希望在流程中允许临时超出边界。

### 2.8 关键点增强
计算机视觉任务，例如人体姿估计、人脸检测和情绪识别，通常使用图像上的关键点。      
在姿态估计中，关键点标记人体关节，例如肩关节、肘关节、腕关节、膝关节等。     

关键点标注以及关键点之间的可视化边缘。图像来自 [COCO数据集](https://cocodataset.org/#home)    
在人脸检测中，关键点标记面部的重要区域，例如眼睛、鼻子、嘴角等。     

[面部关键点,数据来源](https://www.kaggle.com/c/facial-keypoints-detection) 上的面部关键点检测竞赛。      
要定义一个关键点，通常需要两个值： 关键点的x坐标和y坐标。关键点的坐标是相对于图像左上角计算的。

#### 2.8.1 支持的关键点坐标格式
* xy: [x, y]坐标
* yx: [y, x]坐标
* xya: [x, y, angle]坐标
* xys: [x, y, scale]坐标
* xyas: [x, y, angle, scale]坐标
* xysa: [x, y, scale, angle]坐标
* xyz: [x,y, z]坐标

#### 2.10.2 数据卷积
卷     
Albumentations 需要以 numpy 数组的形式接收以下格式的 3D 体积数据：     
* (D, H, W)- 单通道容积（例如，CT扫描）
* (D, H, W, C)- 多通道容积（例如，多模态 MRI）

解释：
* D = 深度（切片数）
* H = 高度
* W = 宽度
* C = 通道数（可选）

3D面具     
分割掩模应与体积尺寸相匹配：
* (D, H, W)- 二元或单类掩码
* (D, H, W, C)多类别口罩      

应用增强功能时，请使用mask3d关键字参数传递这些掩码。

#### 2.10.4 可用的3D变换
* CenterCrop3D: 裁剪3D体积的中心部分
* RandomCrop3D: 随机裁剪3D体积的一部分
* Pad3D: 3D立体垫 
* PadIfNeeded3D: 如果音量小于所需音量，则填充
* CoarseDropout3D: 3D立方体区域的随机丢弃
* 立方对称性: 应用随机立方对称变换

#### 2.10.5 结合二维和三维变换
可在同一流程中组合使用2D和3D变换。当包含2D变换时，albumentations会在每次调用时对其随机参数进行一次采样，并将其以相同的方式应用于深度轴上的每个XY切片。
这确保了翻转或颜色调整等变换在不同切片间的一致性。

```
def create_3d_pipeline(crop_size=(64, 128, 128), p_spatial=0.5, p_intensity=0.3):

            hole_height_range=(0.1, 0.2),
            hole_width_range=(0.1, 0.2),
            p=p_intensity

    ])
```

   * 在生产训练中移除种子
   * 如果需要获得可比较的结果，请始终使用固定种子进行验证/测试转换

   * 将种子值记录到您的实验跟踪系统中

   * 为这两种情况分别创建测试用例

   * 种子会在工人重生时自动更新

   * 更改num_workers增强序列会改变增强序列

```
import albumentations as A
from albumentations.core.transforms_interface import ImageOnlyTransform
import numpy as np

class PerImageNormalize(ImageOnlyTransform):
    """Normalizes an image using its own mean and standard deviation."""

    def __init__(self, p=1.0):
        super().__init__(p=p)

    def get_params_dependent_on_data(self, params, data):
        img = data["image"]
        mean = np.mean(img, axis=(0, 1))
        std = np.std(img, axis=(0, 1))
        std = np.where(std < 1e-6, 1e-6, std)
        return {"mean": mean, "std": std}

    def apply(self, img, mean, std, **params):
        img = img.astype(np.float32)
        return ((img - mean) / std).astype(np.float32)
```

🟡 第三步：高效处理多张图片（中级）
---
目标：使用序列/批次中每张图像自身的均值和标准差高效地对其进行归一化，同时仍然继承自ImageOnlyTransform。
```
import albumentations as A
from albumentations.core.transforms_interface import ImageOnlyTransform
import numpy as np

class PerImageNormalizeEfficient(ImageOnlyTransform):
    """
    Normalizes an image or a sequence/batch of images using per-image
    mean and standard deviation. Overrides apply_to_images for efficiency.
    """
    def __init__(self, p=1.0):
        super().__init__(p=p)

    def get_params_dependent_on_data(self, params, data):
        params_out = {}
        if "image" in data:
            img = data["image"]
            params_out["mean"] = np.mean(img, axis=(0, 1))
            params_out["std"] = np.std(img, axis=(0, 1))
        elif "images" in data:
            imgs = data["images"]
            params_out["mean"] = np.mean(imgs, axis=(1, 2))
            params_out["std"] = np.std(imgs, axis=(1, 2))
        else:
            return {}

        params_out["std"] = np.where(params_out["std"] < 1e-6, 1e-6, params_out["std"])
        return params_out

    def apply(self, img, mean, std, **params):
        img = img.astype(np.float32)
        return ((img - mean) / std).astype(np.float32)

    def apply_to_images(self, images, mean, std, **params):
        images = images.astype(np.float32)

        if images.ndim == 4:
            mean = mean[:, np.newaxis, np.newaxis, :]
            std = std[:, np.newaxis, np.newaxis, :]
        elif images.ndim == 3:
            mean = mean[:, np.newaxis, np.newaxis]
            std = std[:, np.newaxis, np.newaxis]

        return ((images - mean) / std).astype(np.float32)

```

🟠 第四步：影响多个目标的几何变换（高级）
---
目标：创建一个变换，随机移动图像及其所有关联目标，并手动实现目标逻辑。
```
import albumentations as A
from albumentations.core.transforms_interface import DualTransform
import numpy as np
import cv2

class RandomShiftMultiTargetManual(DualTransform):
    """
    Randomly shifts the image and associated targets (mask, bboxes, keypoints)
    with manual implementation of target transformations, respecting internal formats.
    Processes arrays of bboxes and keypoints.
    """
    def __init__(self, x_limit=0.1, y_limit=0.1,
                 interpolation=cv2.INTER_LINEAR,
                 mask_interpolation=cv2.INTER_NEAREST,
                 border_mode=cv2.BORDER_CONSTANT,
                 fill=0,
                 fill_mask=0,
                 p=0.5):
        super().__init__(p=p)
        self.x_limit = x_limit
        self.y_limit = y_limit
        self.interpolation = interpolation
        self.mask_interpolation = mask_interpolation
        self.border_mode = border_mode
        self.fill = fill
        self.fill_mask = fill_mask

    def get_params_dependent_on_data(self, params, data):
        height, width = data["image"].shape[:2]

        dx = self.py_random.uniform(-self.x_limit, self.x_limit)
        dy = self.py_random.uniform(-self.y_limit, self.y_limit)

        x_shift = int(width * dx)
        y_shift = int(height * dy)
        matrix = np.float32([[1, 0, x_shift], [0, 1, y_shift]])
        return {
            "matrix": matrix,
            "x_shift": x_shift,
            "y_shift": y_shift,
            "height": height,
            "width": width,
        }

    def apply(self, img: np.ndarray, matrix: np.ndarray, height: int, width: int, **params):
        return cv2.warpAffine(
            img, matrix, (width, height),
            flags=self.interpolation, borderMode=self.border_mode, borderValue=self.fill
        )

    def apply_to_mask(self, mask: np.ndarray, matrix: np.ndarray, height: int, width: int, **params):
        return cv2.warpAffine(
            mask, matrix, (width, height),
            flags=self.mask_interpolation, borderMode=self.border_mode, borderValue=self.fill_mask
        )

    # Correct method name and signature for multiple bounding boxes
    def apply_to_bboxes(self, bboxes: np.ndarray, x_shift: int, y_shift: int, height: int, width: int, **params):
        """
        Applies shift to an array of bounding boxes.
        Assumes bboxes array is in internal normalized format [:, [x_min, y_min, x_max, y_max, ...]].
        Returns bboxes array in the same format.
        """
        # Calculate normalized shifts
        norm_dx = x_shift / width
        norm_dy = y_shift / height

        # Apply shifts vectorized
        bboxes_shifted = bboxes.copy()
        bboxes_shifted[:, [0, 2]] = bboxes[:, [0, 2]] + norm_dx
        bboxes_shifted[:, [1, 3]] = bboxes[:, [1, 3]] + norm_dy

        return bboxes_shifted

    # Correct method name and signature for multiple keypoints
    def apply_to_keypoints(self, keypoints: np.ndarray, x_shift: int, y_shift: int, **params):
        """
        Applies shift to an array of keypoints.
        Assumes keypoints array is in internal format [:, [x, y, angle, scale, ...]] where x, y are absolute pixels.
        Returns keypoints array in the same format.
        """
        keypoints_shifted = keypoints.copy()
        keypoints_shifted[:, 0] = keypoints[:, 0] + x_shift
        keypoints_shifted[:, 1] = keypoints[:, 1] + y_shift

        return keypoints_shifted

```
🟠 自定义目标路径规划 apply_to_<key>（高级）
--- 
目标：创建一个随机裁剪变换，用于裁剪图像并更新存储在普通字典中的相机内参。关键在于确保裁剪后的图像apply和相机apply_to_camera接收到的裁剪坐标相同。
```Python
import albumentations as A
from albumentations.core.transforms_interface import DualTransform
import numpy as np
from typing import Any

class RandomCropWithIntrinsics(DualTransform):
    """
    Randomly crops an image and updates camera intrinsics that share
    the same sampled crop geometry.
    """

    def __init__(self, crop_height: int, crop_width: int, p: float = 1.0):
        super().__init__(p=p)
        self.crop_height = crop_height
        self.crop_width = crop_width

    @property
    def targets(self) -> dict[str, Any]:
        return {
            **super().targets,
            "camera": self.apply_to_camera,
        }

    def get_params_dependent_on_data(self, params: dict[str, Any], data: dict[str, Any]):
        image_height, image_width = data["image"].shape[:2]

        if self.crop_height > image_height or self.crop_width > image_width:
            raise ValueError(
                f"Crop size ({self.crop_height}, {self.crop_width}) must be smaller than "
                f"or equal to image size ({image_height}, {image_width})."
            )

        x_min = self.py_random.randint(0, image_width - self.crop_width)
        y_min = self.py_random.randint(0, image_height - self.crop_height)
        x_max = x_min + self.crop_width
        y_max = y_min + self.crop_height

        return {
            "x_min": x_min,
            "y_min": y_min,
            "x_max": x_max,
            "y_max": y_max,
        }

    def apply(
        self,
        img: np.ndarray,
        x_min: int,
        y_min: int,
        x_max: int,
        y_max: int,
        **params: Any,
    ) -> np.ndarray:
        return img[y_min:y_max, x_min:x_max]

    def apply_to_camera(
        self,
        camera: dict[str, float],
        x_min: int,
        y_min: int,
        **params: Any,
    ) -> dict[str, float]:
        camera = camera.copy()

        # Cropping changes the coordinate origin. Focal lengths do not change.
        camera["cx"] = camera["cx"] - x_min
        camera["cy"] = camera["cy"] - y_min
        camera["width"] = self.crop_width
        camera["height"] = self.crop_height

        return camera

# --- Usage ---
transform = A.Compose([
    RandomCropWithIntrinsics(crop_height=240, crop_width=320, p=1.0),
], seed=137)

image = np.zeros((480, 640, 3), dtype=np.uint8)
camera = {
    "fx": 500.0,
    "fy": 500.0,
    "cx": 320.0,
    "cy": 240.0,
    "width": 640,
    "height": 480,
}

result = transform(image=image, camera=camera)

cropped_image = result["image"]
updated_camera = result["camera"]

print(cropped_image.shape)
print(updated_camera)

```
